# Angelfire Tab Necromancer — Hyperlocal Lore Deepening Pass

## North Star
This is not primarily a joke generator. It is a necromantic archive of almost-forgotten regional guitar bands that maybe existed, reconstructed from riffs, dead mirrors, and forum residue.

Each normal shrine should feel like:
- a band name
- a place + scene
- a little tab or a few chords
- a specific shard of history
- a few human traces around it

## User Direction to Emphasize
- Keep developing the lore of random bands that barely coalesced or were super hyperlocal
- Focus especially on North Carolina and Pacific Northwest scenes
- Almost every shrine should have at least a little tab / a few chords, and with rare exceptions a band name + a little something about the band
- Comments are good, but should support the band myth / memory
- The resurrected shrine area should feel like it is pulling actual old Angelfire/Geocities sites into view inside an archive frame

## Build Plan

### 1. Make band + tab + band note nearly universal
- Rebalance shrine archetype weighting so the default outcome is a medium-depth recovered band shrine, not a stub/dead page.
- For all archetypes except rare ghost-page failures, require:
  - band name
  - at least a short band note / scene note
  - at least one tab fragment OR chord progression
- Stubs/dead pages should be rare and should usually still preserve some musical residue.

### 2. Expand hyperlocal scene lore heavily
Add much more scene-specific source material for:

#### North Carolina
- Carrboro / Chapel Hill / Durham / Raleigh / Greensboro / Charlotte / Hillsborough / Roxboro / Saxapahaw
- Cat's Cradle orbit, VFWs, church basements, student houses, freight-track practice spots, all-ages rooms, coffeeshop boards, college radio, promoter grudges
- Hyperlocal details: parking lots, flyer etiquette, summer humidity, bad bills, borrowed gear, wrong town attribution, rival scenes, local roads/exits

#### Pacific Northwest
- Eugene / Springfield / Salem / Albany / Corvallis / McMinnville / Bandon / Portland outskirts / Vancouver WA spillover
- Skateshop back rooms, wet garages, all-ages rooms, VFWs, DIY halls, college radio bleed, practice lockouts, rainy parking lots, dead burned CDs, miniDV footage, tape mirrors
- Hyperlocal details: i-5 drives, Dutch Bros cups, damp flyers, lockouts behind industrial strips, county fair side stages, dead community portals

Create more:
- band history fragments
- local venue fragments
- rumor fragments
- breakup fragments
- show-memory fragments
- flyer residue lines
- provenance/mirror/archive fragments

### 3. Tune comments toward memory / correction / defense
Comments should more often do one of these:
- defend the band: "you had to see them live"
- correct the record: "this was Carrboro not Durham"
- correct the tab: "second chord is wrong"
- remember a show: "they opened for ___ at the VFW"
- argue about local scene politics
- remember a member / lineup / practice space
- mention a dead mp3, broken mirror, or old board thread

Keep some dumb junk, but make it subordinate to the sense that the band briefly mattered to real people.

### 4. Strengthen the archive-viewer feeling in the resurrected shrine area
Make the shrine area feel more like a recovered viewport than a generic card. Possible implementation moves:
- stronger provenance / recovery labels
- subtle "mirror / cached / partially restored" cues
- neighboring dead-site / ring context around the active shrine
- more explicit feeling that this is one page being surfaced from a lost cluster
- clearer distinction between the resurrection frame and the old page inside it

### 5. Reduce over-clever generator signals
Downweight or remove content that distracts from the recovered-band illusion.
- Fewer self-consciously clever invented metrics / framing gimmicks
- Less decorative randomness when it does not strengthen the band myth
- Keep the strongest bits that reinforce artifact-ness

### 6. Rebalance archetypes to favor the best mode
Target weighting:
- 45% recovered band shrine (default medium-depth shrine)
- 25% obsessive local shrine
- 15% forum rip
- 10% tab dump
- 5% ghost / dead stub

The average user should mostly encounter shrines that feel like a small real band page, not a gimmick outcome.

### 7. Increase band-specific continuity within a shrine
Within each generated shrine, band-related details should reinforce each other:
- comments should refer to the same towns, venues, lineup disputes, show memories, or song fragments from the lore
- archive notes and flyer lines should cohere with the same scene cluster
- tab/chords should feel attached to that band's identity

### 8. Preserve musical residue even in damaged pages
Even rare broken pages should usually still keep at least one of:
- 1-2 bars of tab
- a chord progression
- tuning info
- a remembered hook description

The user should almost always leave with some musical artifact in hand.

## Verification Plan
Generate 10+ shrines and verify:
1. At least 8/10 shrines include band name + band note + some tab/chords
2. NC and PNW outputs both show richer hyperlocal detail than before
3. Comments often reference memory/correction/defense, not just random noise
4. Resurrected shrine area reads more like an archive viewport
5. Ghost/stub shrines are rare
6. The best shrines feel like small bands that mattered deeply to a tiny scene
